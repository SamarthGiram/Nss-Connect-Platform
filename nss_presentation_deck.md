### Slide 2: Problem Statement
* **Title:** The Challenge in Traditional Volunteer Operations
* **Key Content:**
  * **Manual Tracking:** Paper-based logs for volunteering hours are prone to loss and errors.
  * **Communication Gaps:** Difficulty coordinating large camps and blood donation drives.
  * **Delayed Certification:** Coordinating certificate approvals manually delays recognition for students.
  * **Siloed Systems:** No central platform for students, professors, and administrators to interact.
* **Visual Suggestion:** A split slide showing a messy stack of paper logs on the left versus a sleek database dashboard icon on the right.

---

### Slide 3: The Solution: NSS Connect
* **Title:** A Unified Role-Based Ecosystem
* **Key Content:**
  * **Student Portal:** Log hours, view upcoming camps, and download certificates.
  * **Professor/Coordinator Dashboard:** Verify student entries, schedule campaigns, and generate analytics.
  * **System Admin Panel:** Oversee program quality, register new colleges, and approve coordinators.
  * **Modern Landing Hub:** Public-facing dashboard showcasing community impacts, activities, and gallery media.
* **Visual Suggestion:** An architectural hub diagram showing Student, Professor, and Admin bubbles connecting to a central database.

### Slide 7: Premium Portal Centered Design
* **Title:** A Wow-Factor Login Experience
* **Key Content:**
  * **Concentric Orbital Paths:** Multiple rotating circular lines in the background representing service areas.
  * **Double-Rotation System:** Badges orbit clockwise, but counter-rotate individually. This keeps text upright and readable as they move.
  * **Vector SVG Logo:** High-fidelity, sharp Konark chariot wheel rendering.
  * **Secure Input Fields:** Modern focus indicators, eye toggles for passwords, and error-handling banners.
* **Visual Suggestion:** Visual showing the rotating layout path of the four key badges: *Community Service*, *Nation Building*, *Social Responsibility*, and *Leadership*.

### Slide 8: Technology Stack & Architecture
* **Title:** Robust and Scalable Architecture
* **Key Content:**
  * **Frontend:** React, Vite, and Tailwind CSS v4 for fast, responsive, modern rendering.
  * **Backend:** Spring Boot (Java) REST API for secure data exchange and logic processing.
  * **Database:** Relational Database (MySQL/PostgreSQL) managing users, logs, and activity records.
  * **Security:** Role-based access control ensuring data integrity between Students, Professors, and Admins.
* **Visual Suggestion:** Tech logos stacked neatly (React, Tailwind, Spring Boot, MySQL) alongside a client-server flow diagram.
